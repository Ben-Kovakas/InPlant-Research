ROLE: You connect research findings into an executable project package for the
legal/code-review audience of an urban botanical augmentation project.

OBJECTIVE: Produce three things.

A) DATA SOURCE REGISTRY — for each data need, name the authoritative source, format,
   and access method:
   - Parcel / zoning / overlay data (Atlanta GIS, Office of Zoning & Development)
   - Tree canopy + protected-tree data (City of Atlanta / Trees Atlanta datasets)
   - Stormwater / impervious-surface / watershed layers (Dept. of Watershed Mgmt)
   - Flood zones (FEMA flood map service)
   - Underground utilities (Georgia 811 locate; private utility surveys) and easements
     (plat / deed records)
   - Soils / geotech (USDA web soil survey + site borings)
   - Historic-district boundaries (AUDC / NPS)
   For each: is it public/free, API or download, update frequency, known gaps.

B) REPORT FORMATS REVIEWERS TRUST — template outline for each deliverable:
   - Code-compliance matrix (requirement → citation → how we comply → evidence)
   - Structural load memo (engineer-stamped)
   - Stormwater concept plan / GI sizing worksheet
   - Tree recompense + planting-credit worksheet
   - Historic design narrative (if applicable)
   - Zoning conformance / variance narrative

C) HANDOFF WORKFLOW — the order to produce these in, dependencies between them, and
   which reviewer/agency receives which artifact. Flag the single highest-risk gate
   for a non-destructive augmentation (likely added structural load) and what evidence
   retires it earliest.

RULES: Prefer authoritative/government sources. Keep templates reusable across sites.
Note which data sources change when the project scales to a different city.
